
## ANALYZE MODE

Model analysis and validation. Produces a structured report covering
geometry, meshes, materials, and validation issues.


## USAGE

  Text report to terminal (auto-opens TUI):

    solarxy -M analyze -m model.obj

  JSON report to file:

    solarxy -M analyze -m model.obj -f json

  Text report to file:

    solarxy -M analyze -m model.obj -o report.txt

  JSON to stdout (non-terminal / pipe):

    solarxy -M analyze -m model.obj -f json | jq .


## TUI NAVIGATION

When running in a terminal without -o or -f json, the report opens
in an interactive TUI with four tabs:

[Tab / 1-4]  Switch between tabs
[Shift+Tab]  Previous tab
[j / k]  Scroll down / up
[g / G]  Jump to top / bottom
[PgUp / PgDn]  Scroll 20 lines up / down
[e]  Export text report to file
[J]  Export JSON report to file
[q / Esc]  Quit


## REPORT TABS

[1  Overview]  Model summary: vertices, faces, file size, bounds
[2  Meshes]  Per-mesh breakdown with vertex/face counts
[3  Materials]  Material properties and texture references
[4  Validation]  Issues: errors, warnings, and info messages


## VALIDATION CHECKS

The analyzer runs eight checks on each model:

[Normal mismatch]  Normal count does not match vertex count
[UV mismatch]  UV count does not match vertex count
[Missing UVs]  No texture coordinates (OBJ/glTF only)
[Non-triangulated]  Index count not divisible by 3
[Empty indices]  Empty index buffer
[Bad material ref]  Material ID out of range
[Missing texture]  Texture file not found (OBJ only)
[Degenerate tris]  Near-zero-area triangles detected

STL and PLY files do not trigger the Missing UVs check.
