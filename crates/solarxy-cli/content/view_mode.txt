
## VIEW MODE

Real-time 3D model viewer with Cook-Torrance PBR rendering, shadow mapping,
normal mapping, IBL (diffuse + specular), SSAO, bloom, and a 3-light system.

Launch with:  solarxy -m model.obj
The model path is optional — you can also drop a file onto the window.


## DISPLAY

[W]  Cycle view mode (Shaded > Shaded+Wire > Wireframe > Shaded)
[S]  Switch to Shaded mode
[X]  Toggle Ghosted mode
[Shift+W]  Cycle wireframe line weight (Light > Medium > Bold)
[N]  Cycle normals display (Off > Face > Vertex > Face+Vertex)
[U]  Cycle UV overlay (Off > Gradient > Checker)
[B]  Cycle background (White > Gradient > Dark Gray > Black)
[Shift+B]  Cycle bounding box display (Off > Model > Per Mesh)
[Shift+M]  Toggle bloom
[Shift+O]  Toggle SSAO (screen-space ambient occlusion)
[G]  Toggle grid
[A]  Toggle axis gizmo
[Shift+A]  Toggle local axes (model/mesh centers)
[V]  Toggle turntable auto-rotation
[I]  Toggle IBL (image-based lighting)
[Shift+T]  Cycle tone mapping (None > Linear > Reinhard > ACES Filmic)
[Shift+I]  Cycle IBL mode (Diffuse / Full)
[E / Shift+E]  Increase / decrease exposure


## CAMERA

[P]  Perspective projection
[O]  Orthographic projection
[H]  Home — frame model in view
[T]  Top view
[F]  Front view
[L]  Left view
[R]  Right view
[Arrow keys]  Camera movement


## INSPECTION MODES

Number keys switch the active pane to a visualization mode.
In split view, inspection modes apply to the active pane only.

[1]  Shaded — full PBR rendering (default)
[2]  Material ID — flat color per material slot
[3]  UV Map — toggle 2D UV-space view (pan/zoom only)
[4]  Texel Density — blue/green/red heat map of UV density
[5]  Depth — linearized depth buffer (white = near, black = far)

Inspection modes compose independently with view modes (W/X).


## SPLIT VIEWPORT

[F1]  Single viewport (default)
[F2]  Vertical split (left: UV Map, right: 3D)
[F3]  Horizontal split (top: UV Map, bottom: 3D)
[Ctrl+L]  Toggle camera linking between panes

Each pane has its own camera and display settings. The active pane
is determined by cursor position and highlighted with a blue border.


## UV MAP

When a pane is in UV Map mode (press 3):

[U]  Cycle UV background (Dark > Checker > Texture)
[O]  Toggle UV overlap display
[Shift+W]  Cycle wireframe line weight

Left/middle mouse drag to pan, scroll to zoom.
UV overlap regions are highlighted in semi-transparent red.


## OTHER

[C]  Capture screenshot (PNG)
[Shift+S]  Save current settings as preferences
[Shift+L]  Toggle lights lock (lights follow camera)
[Shift+V]  Toggle validation overlay
[Tab]  Toggle sidebar panel
[?]  Toggle keyboard hints overlay
[Esc]  Exit viewer


## MOUSE

[Left drag]  Orbit camera
[Middle drag]  Pan camera
[Scroll wheel]  Zoom in / out


## DRAG & DROP

Drop a model file onto the viewer window to load it.
Dropping a new file mid-session replaces the currently loaded model.
Drop an .hdr or .exr file to use it as an IBL environment map.


## VIEW MODES

[Shaded]  Full PBR shading with materials and lighting
[Shaded+Wire]  PBR shading with wireframe overlay
[Wireframe]  Wireframe only, no shading
[Ghosted]  Semi-transparent ghosted view


## SIDEBAR PANEL

Press Tab to toggle the sidebar. It provides interactive controls
grouped into collapsible sections:

[View]  View mode, inspection mode, normals, UV, background, bounds
[Display]  Grid, axes, turntable, lights lock
[Validation]  Show/hide validation overlay, issue list
[Post-Processing]  Bloom, SSAO, tone mapping, exposure
[Lighting]  IBL mode

All controls are bidirectionally synced with keyboard shortcuts.


## HEADS-UP DISPLAY

The HUD overlays real-time information on the viewer window.

[Top-left]  Model filename, mesh count, polygon/triangle/vertex
              counts, bounding box dimensions (when active),
              frame time and FPS.

[Top-right]  Non-default render states: view mode, projection,
              normals, background, wireframe weight, grid,
              lights lock, axes, bounds, IBL mode, SSAO,
              tone mapping.

[Bottom]  Keyboard hints bar (toggle with ?).

Toast notifications appear briefly to confirm actions (green for
success, red for errors).
